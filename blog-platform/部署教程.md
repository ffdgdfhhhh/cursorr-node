# 创享社区：服务器部署教程（阿里云 ECS + 宝塔）

本文说明如何将 **Vue 前端 + Node 后端 + MySQL** 的项目部署到 Linux 服务器（以 **宝塔面板** 为例），并汇总常见问题。

---

## 一、架构说明

| 组件 | 说明 |
|------|------|
| 前端 | Vue 3 + Vite，构建产物在 `client/dist` |
| 后端 | Node.js（Express + Sequelize + Socket.io），入口 `server/app.js`，默认端口 **3000** |
| 数据库 | MySQL 8（宝塔安装），库与用户需在面板中创建 |

生产环境推荐：**Nginx（80/443）反向代理** → **本机 `127.0.0.1:3000`**；或使用宝塔「整站反代」到 3000（由 Node 同时提供静态页与 API）。

---

## 二、服务器准备

### 2.1 安全组（阿里云控制台）

入方向放行：**22**（SSH）、**80**、**443**；宝塔端口（常见 **8888**，以安装时提示为准）。

### 2.2 宝塔软件

在 **软件商店** 安装：**Nginx**、**MySQL 8**、**Node 版本管理器**（建议 Node **18+**）、**PM2 管理器**（或全局安装 `pm2`）。

### 2.3 数据库

在 **数据库** 中新建库与用户（例如库名、用户名均为 `cursorsql`），记下密码。  
项目首次启动会用 Sequelize **自动建表**，一般**不必**导入根目录的 `database.sql`（可选参考）。

---

## 三、本地打包（Windows / Mac）

### 3.1 构建前端

```bash
cd client
npm install
npm run build
```

生成目录：`client/dist`。

**生产环境 API 地址**：推荐使用 `client/.env.production` 中 **`VITE_API_BASE`、`VITE_SOCKET_URL` 留空**，这样请求走**当前访问域名**下的 `/api`，避免仍指向旧 IP 导致 404。Socket 在未配置时使用浏览器当前 `origin`。

### 3.2 部署压缩包（可选）

本仓库打包脚本思路：

- 包含：`server/`（**不含** `node_modules`、`uploads`）、`client/dist/`、`database.sql`、`deploy/nginx.example.conf`
- **不要**依赖压缩包内的 `.env` 上线（易覆盖服务器已验证的配置）；若必须打入 `.env`，解压后务必核对数据库账号与域名

在服务器 **`server` 目录**执行：

```bash
npm install --omit=dev
```

## 四、上传到服务器与目录结构

建议站点根目录类似：

```text
/www/wwwroot/你的域名/
└── blog-platform/
    ├── database.sql          # 可选
    ├── deploy/
    ├── client/
    │   └── dist/             # Nginx root 或 Node 静态目录
    └── server/               # PM2 工作目录
        ├── app.js
        ├── package.json
        ├── .env              # 仅在服务器上创建，勿提交仓库
        └── …
```

宝塔新建站点时目录里的默认 `index.html` 等可删可留；只要 **网站运行目录 / Nginx root** 指向 **`blog-platform/client/dist`**（整站反代到 Node 时则由 Node 提供 `dist`，见下文）。

**上传 zip 后必须在面板或 SSH 中解压**；仅上传 zip 不解压不会更新网站。

## 五、后端环境变量 `server/.env`

在 **`…/blog-platform/server/.env`** 新建（可从 `.env.example` 复制），示例：

```env
PORT=3000
NODE_ENV=production
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=cursorsql
DB_USER=cursorsql
DB_PASSWORD=宝塔中该数据库密码
JWT_SECRET=足够长的随机字符串（切勿使用弱密钥）
CLIENT_ORIGIN=http://你的域名
```

说明：

- **`CLIENT_ORIGIN`** 必须与浏览器访问地址一致（含协议）；上 HTTPS 后改为 `https://你的域名`
- **`JWT_SECRET`** 用于签发登录 Token，泄漏会导致严重安全风险
- 若日志提示 **「未找到 .env」**，说明路径不对或文件名错误（不能是 `.env.txt`）

无 **`nano`** 时可用 **`vi .env`**，或使用：

```bash
cat > .env << 'EOF'
…
EOF
```

## 六、安装依赖并启动后端

```bash
cd /www/wwwroot/你的域名/blog-platform/server
npm install --omit=dev
```

### PM2（推荐）

```bash
pm2 start app.js --name chuangxiang
pm2 save
pm2 startup    # 按屏幕提示执行一次，实现开机自启
```

修改 `.env` 后：

```bash
pm2 restart chuangxiang
# 若 PM2 单独配置了环境变量，可用：pm2 restart chuangxiang --update-env
```

自检：

```bash
curl -s http://127.0.0.1:3000/api/health
```

应返回 JSON，`success` 为 true。

## 七、Nginx / 宝塔网站配置

### 方式 A：整站反向代理到 `3000`

目标 URL：`http://127.0.0.1:3000`，代理目录一般为 **`/`**。  
Node 在存在 `client/dist` 时会托管前端并处理 `/api`、`/uploads`、Socket。

需开启 **WebSocket**（私信），否则在宝塔反向代理「高级」中勾选 WebSocket，或参考 `deploy/nginx.example.conf` 中 `/socket.io/` 段。

### 方式 B：静态 + 按路径反代

- **root**：`…/blog-platform/client/dist`
- **`location /`**：`try_files $uri $uri/ /index.html;`
- **`/api/`、`/uploads/`、`/socket.io/`** → `proxy_pass http://127.0.0.1:3000;`（含 WebSocket 头）

示例片段见：`deploy/nginx.example.conf`（需把 `server_name`、`root` 换成你的实际值）。

修改后：**保存并重载 Nginx**。

## 八、域名与 IP 非常重要的一点

**用域名访问**与**用公网 IP 访问**在 Nginx 里往往是 **两个站点（两个 `server`）**：

- 若只为域名配置了「反代到 3000」，而 **IP 默认站点**仍指向旧静态目录，则通过 IP 打开会看到 **旧前端**（例如仍请求 `http://IP/api/...` 导致 404）。
- 域名若反代到 3000，而后端未启动，会出现 **502 Bad Gateway**。

建议：**对外统一使用域名**；若要让 IP 与域名一致，在宝塔中把 **默认站点 / IP 站点** 也改为同一 **root** 或同一 **反代到 3000**，或对 IP 做 **301 跳转到域名**。

## 九、更新部署流程小结

1. 本地：`cd client && npm run build`
2. 上传 **`client/dist`**（或整包 zip 并解压覆盖）
3. 若覆盖了 **`server`**：**务必**在该目录执行 **`npm install --omit=dev`**
4. **不要轻易覆盖** 服务器上已可用的 **`server/.env`**；若被覆盖，按第五节重新填写数据库与 `CLIENT_ORIGIN`
5. **`pm2 restart chuangxiang`**
6. 浏览器 **强制刷新** 或使用无痕窗口（避免旧 JS 缓存）

## 十、常见问题速查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `Cannot find module 'dotenv'` | 未执行或未成功执行 `npm install` | 在 `server` 目录 `npm install --omit=dev` |
| `未找到 .env` / `using password: NO` | 无 `.env` 或路径错误 | 在 `server` 目录创建 `.env` 并填写 `DB_PASSWORD` 等 |
| `Connection refused :3000` | 后端未监听 | 查 `pm2 status`、`pm2 logs`，修复后重启 |
| 域名 **502** | Nginx 反代正常但上游挂掉 | 同上，保证 `curl 127.0.0.1:3000/api/health` 正常 |
| 控制台仍请求 **`http://IP/api/...` 且 404** | 浏览器在用旧 `dist` 或用 IP 打开旧站点 | 更新 `dist`、用域名访问、清缓存；确认 IP 站点配置 |
| API 正常但页面空白或路由 404 | Nginx 未 `try_files` 回退 `index.html` | 检查 SPA 的 `location /` 配置 |

---

## 十一、安全建议

- 生产环境使用强 **`JWT_SECRET`**，定期更换数据库密码
- **不要将** 含真实密码的 `.env`、整站备份随意上传到公开位置
- 对外仅开放 **80/443**（及 SSH），不必将 **3000** 暴露到公网（若仅用 Nginx 反代）
